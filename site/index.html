H.323 plugin for Ethereal

Preface

This project started in February 2000, when I started to work for Philips Business Communications in Hilversum, the Netherlands. They asked me to create a network sniffer which was able to decode H.323 traffic, while I was getting practical experience for my study, Computer Science.

After some objective multiple criteria analysis (yeah, right), I decided to focus on Ethereal and OpenH323. I changed the existing Q.931 dissector which was already used in Ethereal, then I also changed (well, rewrote) the existing RTP and RTCP (RFC 1889) dissectors to suit my needs. I changed the existing ASN.1 compiler from the OpenH323 project to generate source to be used in an Ethereal dissector. Since March 2002 the H.323 support is supplied as a plugin for Ethereal instead of a binary.

Since March 2002, this project is available on Sourceforge. If you'd like to have more information, checkout the Sourceforge site.

News

10-06-02
I have received some bug reports:

  • Using the current plugin (0.0.3b) with Ethereal 0.9.4 will result in a console window opening showing file proto.c: line 1061: assertion failed: <hfinfo->type == FT_IPv4>, and Ethereal terminating with a "Runtime Error!"
    The current plugin is NOT useable with Ethereal 0.9.4, use Ethereal 0.9.3. When I have time, I will release a new binary version of the plugin with Ethereal 0.9.4 support. If you have access to MS Visual Studio, you can compile the source yourself. It will work without any changes, just follow the instructions.
  • Some people report that when Ethereal is loading a capture file whith the plugin installed it can crash.
    Instead of installing ethereal-setup-0.9.3.exe, install ethereal-setup-0.9.3-1.exe. This will make your problem go away.

19-04-02
I have removed the old news, it was confusing some people, because sometimes I did have Linux support and other times I didn't. People were requesting it when I already had it and wanted to download it when I didn't.
I added a new preface and moved the CallFlow download also to Sourceforge. I didn't create a new project for it, because it is very close to the plugin. I also added icotoprc, which has nothing to do with the plugin itself, it's just a quick hack for some problem I had, and to make it available to everyone I quickly hacked it together as a release in this project ;-)

25-03-02
I have created a project at Sourceforge. Now you can subscribe to mailinglists, submit bugs and download from Sourceforge. The website will remain at voice2sniff.
