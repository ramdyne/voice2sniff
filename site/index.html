Ethereal binary with H.323 support

Preface

This project was done by me from February to June 2000 at Philips Business Communications in Hilversum. They asked me to create a network sniffer which could decode H.323 traffic. I changed the Q.931 dissector which was already used in Ethereal, I also changed the existing RTP and RTCP (RFC 1889) dissectors to suit my needs. I changed an existing ASN.1 compiler from the OpenH323 project to generate source to be used in Ethereal. I graduated on this project with very high grade and received my diploma :-).

The executable that can be downloaded from this page is capable of decoding H.323 (Voice over IP standard) messages. Unfortunately, due to linking problems, currently I can't get it to work on Linux or any other *nix.

Download

Download the Win32 executable (1.3 megabyte) or the textmode variant.
You'll also need the GTK for windows DLL's (1.1 megabyte).

To get Ethereal working, see the instructions on the Ethereal site.

If you want to capture with Ethereal, you'll also need to install a packet driver.

The source is also available.

Known issues

The biggest issue that I know of has to do with an unfortunate part of the protocol, random ports. It is possible to not only see H.245 traffic going from/to random ports, but also H.225.0 Call Signalling traffic. I have seen this behaviour with gateways, but others have also seen this with terminals. There's no easy solution, but I'm working on it.
